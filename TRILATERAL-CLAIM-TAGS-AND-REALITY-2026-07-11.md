## Formula versus observation

An algorithm can be mathematically correct without being live, and a live measurement can be real
without proving a universal theorem. The catalog must retain both dimensions:

```text
mathematical status:
  CANON / PROVED-FOR-DECLARED-DOMAIN / MODEL / FALSE

execution status:
  MEASURED_ACER / MEASURED_LIRIS / MEASURED_THIRD_SEAT / MEASURED_CI /
  MEASURED_REPO / UNVERIFIED_LIVE / CANNOT_SEE
```

Never collapse these into one “real/not real” bit.

## Trilateral minimum

Acer and Liris remain the bilateral implementation pair. For measurements intended to become shared
cross-repository anchors, add an independent third execution seat when practical. Immutable-head CI
is an additional verifier, not a replacement for physical-vantage evidence.

## Tag rules

- `MEASURED_ACER`: Acer executed or directly observed the result.
- `MEASURED_LIRIS`: Liris independently executed or observed it.
- `MEASURED_THIRD_SEAT`: unrelated environment/toolchain executed it.
- `MEASURED_CI`: declared workflow executed the immutable commit.
- `MEASURED_REPO`: deterministic implementation/test exists in the repository.
- `REPOSITORY_REPORTED_TRAINING`: metric appears in source/manifests but was not rerun in this pass.
- `OPERATOR_REPORTED_MEASURED`: exact operator-supplied run; preserve provenance and seek durable receipt.
- `AUDITED`: code/test/doc reading, not execution.
- `CANNOT_SEE`: this vantage lacks access; not a refutation.
- `CONFLICT_KEPT`: evidence disagrees; preserve both.
- `SUPERSEDED`: newer measured mechanism replaces stale prose.
- `UNVERIFIED_LIVE`: design/implementation exists, named deployment not observed.

## No-deflate/no-inflate equation

```text
truth = mechanism + declared scope + provenance + vantage + time + authority
```

Removing scope inflates. Removing provenance deflates. Removing vantage creates false conflicts.
Removing time flattens the old/new migration history. Removing authority confuses verification with
permission.

## Merge rule

Merge additive formulas, receipts, provenance corrections, and supersession notes when conflict-free
and check-clean. Hold only actual failures, secrets, destructive actions, authority changes, or
unsupported stronger claims.
