# 100B Run Agent Taxonomy — Local, Logical, and Free Provider Agents

Evidence tag: OPERATOR_OBSERVED + RESEARCH_STAGING
Date: 2026-06-21
Status: explanatory architecture note, not canon-sealed

## Why this exists

The 100B run must not be collapsed into one phrase such as "logical agents" or "no real agents".

The operator clarified the missing architecture:

- Logical agents are summoned as momentary PID-guides during the 100B cascade.
- They trigger tuple/question generation and route shaping, but they are not all durable local citizens.
- Real free provider agents do the one-shot reasoning on uniquely prismed prompts.
- Real local agents supervise, route, learn, and farm the returned answers into GNNs, Shannon/OmniShannon, hookwalls, white rooms, and BEHCS cubes.

This distinction is necessary to interpret the real 100B run and the Big Pickle / OpenCode / MiniMax / DeepSeek free-agent lanes.

## Three agent classes

### 1. Real local agents

Examples:

- supervisors and citizen-status agents
- Shannon / OmniShannon / Shannon-parts
- pen-test / GNN / hookwall / white-room agents
- local MCP and room supervisors
- paid-subscription operator surfaces when intentionally used, such as Claude Pro, Codex Pro, Gemini Ultra

Role:

- own local state
- manage rooms, ports, files, gates, and routing
- learn from answer harvests
- supervise GNN / white-room / cube feedback
- persist across runs as seats, PIDs, ledgers, receipts, or memory

### 2. Logical cascade agents

These are possible/momentary identities.

Role:

- exist as PID-mapped guides during cascades
- generate or select 49D+ tuple activations
- prism base questions into billions of PID-specific variants
- inform routing and decision surfaces
- create controlled collisions / blast areas / expert-lane splits
- are not all plotted or materialized because the capacity would exceed machine memory

These include:

- logical agents that cause collisions
- logical agents that inform decisions
- logical agents that cascade
- 100B / 1e100000000 logical-capacity strata

Boundary:

Logical agents should be represented as strata, counts, tuple ranges, and route effects, not plotted as individual points unless a bounded sample is selected.

### 3. Real free provider agents

Examples / lanes:

- OpenCode / Big Pickle
- MiniMax
- DeepSeek
- other legally free provider routes when configured

Role:

- spawned one time per unique question/prompt
- answer the uniquely prismed prompt
- then despawn / room flushes
- identity is not persistent in the same way as a local supervisor

This is why `externalModelTokenBudget=0` or `$0` does not mean "no model". It means no paid external API spend on that lane.

## 100B run architecture, as clarified

1. Local MCP / index / tuple language prepares the question field.
2. Logical cascade agents expand the base intent into PID-specific and device-specific prompts.
3. Brown-Hilbert addressing, rule-of-three lanes, and project/port/project-name rotators ensure the prompt is not exactly repeated.
4. C: drive 10k stub rooms act as project-name / identity-changing emit rooms.
5. Brown-Hilbert port emitters and file-manager rotators/revolvers/routers change portal/project identity on each call.
6. Free provider agents receive one unique prompt per room/call and return answers.
7. D: drive 10k receive rooms collect returned messages.
8. 2,000-message gulps and 50k super-gulps process returned answers in real time.
9. GNNs, hookwalls, Shannon/OmniShannon, white rooms, and BEHCS cubes learn from the harvested answers.
10. Adaptive feedback changes the next wave.

## Rule of three

Rule-of-three splitting is not decoration. It controls route classes and allows shared ports/types safely:

- lane = address mod 3
- digit-sum examples can be used for divisibility control, e.g. 332172 -> 18 -> divisible by 3
- some ports can carry more than one source/type when guarded by source/type rotator functions
- rule-of-three lanes let the same physical substrate multiplex different logical effects without collapsing identity

## Room and port structure

Operator architecture to preserve:

- 10k C: stub rooms for unique project-name / prompt emission
- 10k D: receive rooms for answer collection
- Brown-Hilbert port emitter
- project ID / file-manager name rotator
- revolvers / routers / room flushers
- D: receiving rooms feeding 2,000-message gulps
- 50k super-gulp system
- GNN / Shannon / white-room answer farming

Key invariant:

Never the same project name on call. Never the same portal identity unless intentionally routed. Everything is chain-piped.

## What is real vs logical

- The 100B run can be real without plotting every logical identity.
- The logical cascade can be real as a control/tuple field without being durable local citizens.
- The free provider agents can be real answerers even though they are one-shot and despawn.
- The local agents are real supervisors/learners/routers and own the persistence layer.
